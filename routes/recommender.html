Inheritance is when a 'class' derives from an existing 'class'.So if you have a Person class, then you have a Student class that extends Person, Student inherits all the things that Person has.There are some details around the access modifiers you put on the fields/methods in Person, but that's the basic idea.For example, if you have a private field on Person, Student won't see it because its private, and private fields are not visible to subclasses.Polymorphism deals with how the program decides which methods it should use, depending on what type of thing it has.If you have a Person, which has a read method, and you have a Student which extends Person, which has its own implementation of read, which method gets called is determined for you by the runtime, depending if you have a Person or a Student.It gets a bit tricky, but if you do something likePerson p = new Student();p.read();the read method on Student gets called.Thats the polymorphism in action.You can do that assignment because a Student is a Person, but the runtime is smart enough to know that the actual type of p is Student.Note that details differ among languages.You can do inheritance in javascript for example, but its completely different than the way it works in Java.

3 Text :

Polymorphism: The ability to treat objects of different types in a similar manner.Example: Giraffe and Crocodile are both Animals, and animals can Move.If you have an instance of an Animal then you can call Move without knowing or caring what type of animal it is.Inheritance: This is one way of achieving both Polymorphism and code reuse at the same time.Other forms of polymorphism:There are other way of achieving polymorphism, such as interfaces, which provide only polymorphism but no code reuse (sometimes the code is quite different, such as Move for a Snake would be quite different from Move for a Dog, in which case an Interface would be the better polymorphic choice in this case.In other dynamic languages polymorphism can be achieved with Duck Typing, which is the classes don't even need to share the same base class or interface, they just need a method with the same name.Or even more dynamic like Javascript, you don't even need classes at all, just an object with the same method name can be used polymorphically.

Query Analysis

Explanation

Crawling

I have used scrapy library in python to crawl Java docs as well as Java Oracle Tutorial website. Data is crawled by 2 spiders (one for JavaWikiBook and another for Java Oracle TM tutorials) :
  • For JavaWikiBook - all headings are being crawled, then the textual data as well as the code snippets for each heading is being crawled.
    No. of webpages crawled: more than 187
  • For Java Oracle TM tutorials - all the pages are being crawled on the basis of page-content div.
    No. of webpages crawled: more than 2000
After the data is crawled it is passed through elastic search pipeline to get indexed.

Indexing

In order to index the data crawled from websites I have used a Lucene based library ElasticSearch It works in the following way :
  • Documents are indexedstored and made searchableby using the index API of ElasticSearch and then passed through a custom analyzer
  • The data that comes from the crawled website is first tokenized (breaking into words) using standard tokeniser of ElasticSearch. The standard tokenizer provides grammar based tokenization (Unicode Text Segmentation algorithm) and works well for most languages.
  • The tokenized data is then stemmed i.e words are reduced to their root form. The algorithm used by elastic search to stem the data is Porter Stemming algorithm
  • The stemmed data is further used to remove stop words from it. Filtering stop words help improving accuracy of content indexed or searched. To remove stop words we have used english library where ElasticSearch defines a pre set of words that can be removed.
  • The tokens are then stored in an index file where each token is associated with the documents that contain it.
  • The query that we search for is then passed through the same analyser to build query tokens and then used to look the index data to get the recommended data

Originality

  1. I have implemented a search bar> to search for a customized query other than the given queries
  2. This search bar provides 2 options to search for a query in a given Java Wikibook or Java Oracle TM tutorials.
  3. Apart from this for each given query I have shown an option that shows how a query is processed (Query Processing) before it is used to lookup in a document
  4. For the given query user is also given an option to either view the section of the recommended web page, scrapped content or code snippets
  5. For each post to view recommended results, a pagination is done in pop up box to view per recommendation per query per page

Similarity Measure

ElasticSearch uses BM25 similarity measure by default. This is a TF/IDF based similarity that has built-in tf normalization and is supposed to work better for short fields. BM25 is a bag-of-words retrieval function that ranks a set of documents based on the query terms appearing in each document, regardless of the inter-relationship between the query terms within a document (e.g., their relative proximity). It is not a single function, but actually a whole family of scoring functions, with slightly different components and parameters. Okapi BM25 (BM stands for Best Matching) is a ranking function used by search engines to rank matching documents according to their relevance to a given search query.
